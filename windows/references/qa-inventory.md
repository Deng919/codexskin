# Windows 主题 QA 清单

## 用户可见约束

1. 首页与会话页使用同一套整窗背景和半透明组件体系。
2. 不新增、替换、隐藏或改写 Codex 原生文案与交互。
3. 不创建自定义头像，不在首页叠加重复人物。
4. 建议卡、项目选择器、composer、侧栏、菜单和环境面板保持可点击。
5. Store 安装包、签名、`WindowsApps` 和 `app.asar` 保持不变。
6. 宠物 `avatar-overlay` 页面不注入主题，窗口背景保持透明。

## 自动化检查

- `node --test windows/tests/*.test.mjs` 全部通过。
- `python -m unittest discover -s windows/tests -p "test_*.py" -v` 全部通过。
- `renderer-inject.js` 和 `injector.mjs` 通过 `node --check`。
- PowerShell 脚本解析错误数为 0。
- `git diff --check` 无输出。
- 每个主题资源位于自身目录内，单文件小于 16 MiB。

## 首页检查

分别检查 `1280x800` 和 `1600x900`：

- 背景比例稳定，视觉主体没有被窗口边缘破坏性裁切。
- 左侧标题与人物脸部、眼睛和头部轮廓不发生不可读重叠。
- 原生建议卡为 2 至 4 张响应式卡片，无跳高和横向溢出。
- 项目选择区与 composer 透明度足够，同时文字和焦点状态清晰。
- 页面不存在圆形自定义头像、独立人物横幅或装饰文案。

## 会话页检查

- 背景固定，不随消息滚动。
- 正文、代码、命令、时间戳和状态优先可读。
- 普通消息不被统一包成厚重卡片。
- 环境面板、弹出菜单和附件预览位于主题背景之上。
- composer 保持原生高度、控件顺序和点击区域。

## 切换与恢复

- `switch-dream-skin.ps1` 同步更新 `active-theme.txt`、基础配色和注入守护进程。
- `xuanjia-chijin` 与 `blue-night-red-eyes` 可以双向热切换。
- reload 后主题自动恢复，不重复创建 style 或装饰节点。
- `restore-dream-skin.ps1` 能移除 live 注入；恢复不影响线程、项目、插件、登录和宠物。

## 拒收条件

- 人物脸、头部或肩部被破坏性裁切。
- 背景与人物重复、出现棋盘格、白边或破图。
- 主题色切换后仍残留上一主题的硬编码颜色。
- 原生文案被修改，或装饰层拦截点击。
- 宠物窗口出现黑底。
- 页面出现横向滚动、组件重叠或无法读取的低对比文字。
