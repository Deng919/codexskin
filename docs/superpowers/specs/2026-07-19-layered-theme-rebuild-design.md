# Windows 分层主题重建设计

日期：2026-07-19

## 背景

当前仓库包含五套 Windows 主题：

- `xuanjia-chijin`（玄甲赤金）
- `blue-night-red-eyes`（冷蓝红瞳）
- `neon-sakura-city`（霓虹樱城）
- `frostbyte-game-room`（霜白电竞）
- `celestial-tide`（星海潮汐）

玄甲赤金已经使用“纯背景 + 透明人物”的真实分层结构。冷蓝红瞳及三套新增主题把人物直接烘焙在背景中，无法像玄甲赤金一样独立调整人物位置、大小和窄窗口裁切。本次工作统一五套主题的资源结构，并以玄甲赤金为视觉基准。

## 目标

1. 五套主题都使用独立纯背景和透明人物层。
2. 人物保留原始参考图中的脸、服装、姿势和比例，不通过 AI 重绘。
3. 人物遮挡后的场景由当前 Cockpit 账号生成的背景编辑结果自然补全。
4. 每套主题可以分别配置桌面端和窄窗口的人物大小与位置。
5. 主页和会话页继续使用相同的全窗口场景，不修改 Codex 原生文案和功能。
6. 主题切换、重载恢复、官方配色恢复和旧主题兼容保持有效。

## 非目标

- 不重新设计人物、服装、姿势或主题文案。
- 不增加头像、宠物或额外装饰覆盖层。
- 不把候选图、原始参考图、处理脚本或 QA 截图提交到仓库。
- 不为主页和会话页制作两套不同构图。

## 资源结构

每套主题采用以下结构：

```text
windows/themes/<theme-id>/
├── theme.json
├── hero.jpg 或 hero.png
├── texture.png
└── assets/
    └── <theme-id>-character-cutout.png
```

- `hero`：16:9 的完整纯背景，人物及其前景组已经移除，遮挡区域已补全。
- `character`：带透明通道的 PNG，只包含该主题定义的前景组。
- `texture`：保留现有主题纹理；不创建人物或头像占位文件。

玄甲赤金保留现有背景和透明人物素材，只补齐显式布局配置并重新验证。其余四套重新制作 `hero` 和透明人物 PNG。

## 前景组边界

通用人物层包含人物身体、脸、头发、服装以及与人物直接绑定的手持物、武器和贴身特效。

- 冷蓝红瞳：人物、服装、头发和人物轮廓光。
- 霓虹樱城：人物、服装、头发和手机；窗台与城市属于背景。
- 霜白电竞：人物、服装、头发、手柄、连接线和电竞椅作为一个完整前景组；电脑、音箱、桌子、显示器、植物、墙面和地面属于背景。
- 星海潮汐：人物、服装、头发和贴身水光；海面、天空及远景光效属于背景。
- 玄甲赤金：沿用现有包含人物、服装、头发、披风和武器的透明前景。

## 制作方案

采用混合方案，避免人物身份和姿势漂移：

1. 使用原始参考图进行本地人物分割和精细透明蒙版处理。
2. 人物层始终保留原图像素，不通过生成模型重绘。
3. 使用当前 Cockpit 账号，以原始参考图为编辑目标，仅移除前景组并补全其遮挡后的场景。
4. 若背景结构、视角、光线或画风发生漂移，只重新生成背景。
5. 若人物边缘出现发丝缺失、白边、色边或背景残留，只重做蒙版和边缘处理。
6. 为每套主题生成合成预览，确认分层重组后仍保持原图的视觉语言。

中间文件统一放在：

```text
D:\Cache\Codex-Dream-Skin\layered-theme-rebuild-2026-07-19
```

仓库正式成品保留在：

```text
D:\Apps\Codex-Dream-Skin
```

## 主题配置与渲染

`theme.json` 增加并使用以下布局参数：

- `characterSize`：桌面端人物背景尺寸。
- `characterPosition`：桌面端人物背景位置。
- `characterSizeNarrow`：窄窗口人物背景尺寸。
- `characterPositionNarrow`：窄窗口人物背景位置。

加载器验证这些字段为安全、有限长度的 CSS 背景尺寸和位置值。缺少新字段时使用当前玄甲赤金规则作为默认值，以兼容旧主题包。

渲染器将参数映射为 CSS 变量，由主页和会话页共用：

- `--theme-character-size`
- `--theme-character-position`
- `--theme-character-size-narrow`
- `--theme-character-position-narrow`

桌面端和窄窗口媒体查询分别使用对应变量，不再把所有主题固定为同一右侧位置和高度。

## 主题落位方向

- 玄甲赤金：保持当前站立人物效果。
- 冷蓝红瞳：右侧半身构图，为左侧导航和正文保留空间。
- 霓虹樱城：右下横向坐姿，避免腿部和长发被不自然裁断。
- 霜白电竞：人物与电竞椅作为整体置于右侧偏下，保持坐姿支撑关系。
- 星海潮汐：右侧站姿，保留头发和贴身水光轮廓。

实际数值以真实 Codex 窗口 QA 为准，不在设计阶段锁死像素值。

## 资产质量标准

- 纯背景统一输出为 3840×2160，或在无损视觉质量前提下保持等价 16:9 高分辨率。
- 背景中不得残留人物轮廓、透明空洞、模糊涂抹或不连续结构。
- 人物 PNG 必须为 RGBA，四角透明，主体 alpha 包围盒非空。
- 发丝、衣摆、轮廓光和水光允许合理半透明，不得出现明显白边、色边或原背景块。
- 单个资源必须低于加载器 16 MiB 限制。
- 人物在宽窗口和窄窗口中保持可见，不遮挡主要正文、输入框和关键导航操作。

## 验证

自动验证包括：

- Node 主题加载器和渲染合约测试。
- Python 图片尺寸、格式、透明通道、alpha 包围盒和资源大小测试。
- JavaScript 语法检查、PowerShell 解析检查和 `git diff --check`。
- 所有五套主题均声明有效 `character` 资源，并具有完整布局参数。

真实窗口验证包括：

- 主页和具体会话页。
- 宽窗口和窄窗口。
- 五套主题热切换。
- renderer reload 后主题恢复。
- 官方初始配色恢复。
- 人物与界面文字、导航、时间线和输入框不存在不合理遮挡。

## 失败处理与清理

- 一次只处理一套主题，依次检查纯背景、透明人物和真实窗口合成。
- 未通过视觉检查的候选资源不得替换仓库正式文件。
- 背景问题和人物蒙版问题分开迭代，避免重复生成已经确认的部分。
- 全部主题通过后删除缓存中的候选图、处理脚本和 QA 截图。
- 最终提交只包含正式主题资源、配置、加载器/渲染器改动、测试和必要文档。
- 推送前获取 `publish/main`，不得强推覆盖远端改动。
