# 玄甲赤金 Windows 主题设计规格

## 目标

在保留 Windows 主题包加载器与 Codex 原生功能的前提下，重新制作 `xuanjia-chijin` 视觉层。成品应达到仓库 macOS 示例的组件融合度，而不是简单换色或铺一张壁纸。

主题必须满足以下结果：

- 首页清楚展示人物的脸、头冠、肩甲和上半身，裁切稳定且符合构图审美。
- 任务页仍能感知人物与场景，但人物退为背景，不影响消息、命令输出和控件阅读。
- 侧栏、顶栏、横幅、建议卡、项目选择区、输入框和任务消息共享同一套材质、光源与圆角语言。
- 不新增、替换、隐藏或改写任何 Codex 原生文案。
- Windows 主题的安装、切换、回退和安全边界保持不变。

## 视觉基线

结构与交互直接对齐仓库 `macos/assets/dream-skin.css` 的成熟实现：

- 首页采用约 `252px` 高、`24px` 圆角的主横幅。
- 建议卡采用约 `126px` 高、`21px` 圆角、双层材质和轻微上浮悬停。
- 输入框采用约 `23px` 圆角的悬浮玻璃层，并与项目选择区形成上下两层。
- 任务页使用整页壁纸、左侧阅读遮罩、半透明消息层和悬浮输入区。
- 保留 macOS 示例的几何、层级、留白和动效节奏，但不移植其主题名称、标语、状态、项目标签或引用文案。

视觉材质采用一套统一规则：

- 基底：黑化玄铁和低反射深灰，不使用纯黑平面铺满所有组件。
- 次级材质：暗红漆面，只用于选中态、图标底、局部内衬和细微反光。
- 强调：低饱和旧金，只用于边缘、焦点、图标环和交互反馈，不大面积铺色。
- 光源：统一采用人物原图的右上方冷光，边缘亮度、投影方向和人物光线保持一致。
- 圆角：延续仓库示例的柔和大圆角，不再使用大量割裂的硬边框。

## 图像资产

### 透明人物主资产

正式资产：

`windows/themes/xuanjia-chijin/assets/xuanjia-character-cutout.png`

该文件为 `1672x941` RGBA PNG，人物背景透明。它是唯一人物源资产，用于避免首页、任务页和头像出现不同脸型、服装或画风。

### 首页人物层

首页横幅由三层组成：

1. 无人物的雪山战场底图。
2. 左深右浅的阅读遮罩与玄铁内框。
3. 透明人物主资产，定位在右侧，脸部位于横幅右侧约 `65%` 的视觉焦点。

人物层独立于背景缩放。宽屏时展示头冠、脸、肩甲、胸甲和部分披风；窄屏时优先保留头冠、脸和双肩，不能用 `background-size: cover` 任意裁掉面部。

### 任务页人物层

任务页复用透明人物主资产，不制作第二个身份不一致的人物。人物放在右侧背景层，降低亮度和饱和度，并通过左侧阅读遮罩让正文区域保持高对比。人物不得位于消息正文正下方，也不得因滚动而随单条消息移动。

### 头像资产

从透明人物主资产确定性裁出 `512x512` 的 `xuanjia-avatar.png`：

- 构图包含完整脸部、头冠和龙纹肩甲。
- 背景透明，外围使用细旧金环和暗红内衬。
- 头像只作为主题装饰标记，放在仓库示例“传送门标记”对应的主题 chrome 位置。
- 不替换用户账户头像、模型图标、项目图标或任何具有原生语义的图标。

## 组件设计

### 首页横幅

- 保留 Codex 原生标题、项目选择按钮及其事件。
- 标题在左侧 `55%` 阅读区，人物位于右侧。
- 不使用 CSS `content` 注入主题名、标语、项目提示或其他文字。
- 横幅包含外描边、内描边、低强度金色边缘光和向下投影，所有效果使用相同右上光源。

### 建议卡

- 保留原生四张卡片、图标和文字。
- 卡面为玄铁玻璃双层材质，图标底使用暗红漆，外圈使用旧金。
- 悬停仅上移约 `4px` 并增强边缘光，不改变布局尺寸。
- 卡片之间保持仓库示例的连续节奏，不做孤立的大黑框。

### 项目选择区与输入框

- 沿用仓库示例的双层悬浮关系和圆角比例。
- 项目选择区只显示 Codex 已有内容，不增加“选择项目”等主题标签。
- 输入框保持原生按钮、模式选择、审批状态、模型名称和发送按钮。
- 透明人物或头像不得遮挡输入、下拉菜单、提示文本和焦点环。

### 侧栏与顶栏

- 侧栏使用低反射玄铁材质，顶栏使用相同色温的更薄表面。
- 选中态使用暗红半透明底、细旧金边和轻微内发光。
- 不改侧栏原生项目名、任务名、设置文字或窗口菜单。
- 滚动条保持窄而可见，颜色与旧金强调一致，但不能抢占注意力。

### 任务页

- 壁纸和人物是固定背景层，消息与工具输出仍为实时原生 DOM。
- 正文所在左中区域覆盖深色阅读遮罩；右侧人物可见但对比低于正文。
- 消息容器使用轻度透明和模糊，不把每段文字包成厚重卡片。
- 任务页输入框与首页输入框共享同一材质变量和几何规则。

## 代码边界

- 保留现有 Windows 主题包加载器、`theme.json`、启动脚本、回退机制和 CDP 本机监听策略。
- 视觉重构限制在 `xuanjia-chijin` 主题资产、主题变量和 Windows 注入 CSS/renderer 的主题呈现逻辑。
- renderer 只负责识别首页、任务页和必要的壳状态，不写入可见文字。
- 所有选择器必须有缺失保护；Codex DOM 变化时，主题可以局部失效，但不能阻止应用启动或原生交互。
- 资产缺失时回退到纯材质界面，不显示破图图标或空白遮罩。

## 响应式规则

必须验证 `1280x720`、`1600x900` 和 `1920x1080`：

- `1280x720`：横幅可以降低高度，但人物脸、头冠和双肩仍完整可见；建议卡和输入框不得重叠。
- `1600x900`：作为主设计视口，完整展示首页结构和合理的任务页背景比例。
- `1920x1080`：限制内容最大宽度，人物不能因空间增加而无限放大或偏离焦点。
- 窄视口允许减少披风与下半身展示，但不得先裁脸、头冠或肩甲。

## 验证

自动测试至少覆盖：

- 主题清单与资产路径有效。
- CSS 和 renderer 不包含主题可见文案注入。
- 首页、任务页和输入框选择器存在并有缺失保护。
- 回退和还原不会残留主题资产或状态。

真实应用 QA 至少覆盖：

- 首页横幅、建议卡、项目选择和输入框。
- 普通任务页、长消息、命令输出、图片附件和右侧环境信息面板。
- 侧栏滚动、项目选中态、输入框聚焦、下拉菜单和发送按钮。
- 三个目标视口的截图、文本溢出、组件重叠和人物裁切。

透明人物资产必须叠加在深灰、暗红和浅灰三种底色上检查白边。任何白色棋盘格、硬白边或明显发丝缺口都视为失败。

## 验收标准

- 首页第一眼能识别人物与玄甲赤金主题，同时仍然明确是 Codex 工作界面。
- 组件看起来属于同一产品，不出现“背景图加一组黑框”的割裂感。
- 任务页连续阅读时，人物存在但不与正文争夺注意力。
- 原生文案、按钮、菜单、项目和任务信息与未安装主题时一致。
- 主题关闭或回退后，Codex 恢复到原生外观和行为。
